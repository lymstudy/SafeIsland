# AXI Safety Island — VCS 仿真环境说明

## 1. 环境要求

| 工具 | 版本 | 说明 |
|------|------|------|
| VCS | T-2022.06 及以上 | Verilog/SystemVerilog 仿真 |
| Verdi | T-2022.06 及以上 | 波形查看（可选） |
| urg | 随 VCS 安装 | 代码覆盖率报告 |
| OS | Linux (RHEL/CentOS 7+) | VCS 运行环境 |

## 2. 目录结构

```
submission/
├── scripts/          ← 本目录，运行入口
├── rtl/              ← RTL 源码（safety_island_top.f）
├── tb/
│   ├── functional/   ← 功能仿真 testbench
│   └── fault_injection/  ← 注错仿真 testbench
├── sim/
│   ├── functional/   ← 功能仿真结果与行覆盖率
│   └── fault_injection/  ← 注错结果与诊断覆盖率
│   (运行中间文件在 sim/work/，脚本结束后自动删除)
└── fault_campaign/   ← 注错目标配置 CSV
```

**scripts/ 仅含**：Makefile、run_*.sh、本说明文件。不在此目录保留 build/csrc/waves 等生成物。

## 3. 运行方式

将 `submission` 解压到任意路径后：

```bash
cd <submission>/scripts
```

### 3.1 仅功能仿真（34 case + 行覆盖率）

```bash
bash run_functional.sh
# 或
make functional
```

结果目录：`../sim/functional/`
- 日志：`logs/full_run.log`
- 行覆盖率：`coverage/urgReport/index.html`

### 3.2 仅注错仿真（54 基线 + 610 全量 bit）

```bash
bash run_fault.sh
# 或
make fault-all
```

结果目录：`../sim/fault_injection/`
- 日志：`logs/fault_run.log`、`logs/batch_run.log`
- 报告：`reports/fault_injection_report.csv`
- 诊断覆盖率摘要：`diagnostic_coverage_summary.txt`

### 3.3 全套（功能 + 注错 + 覆盖率）

```bash
bash run_all.sh
# 或
make all
```

## 4. Makefile 目标速查

| 目标 | 说明 |
|------|------|
| `make full` | 功能仿真 34 case |
| `make cov-full` | 功能仿真 + 行/翻转/条件/FSM 覆盖率 |
| `make fault` | 注错基线 54 case |
| `make batch` | 注错全量 bit 扫描 610 case |
| `make functional` | full + cov-full |
| `make fault-all` | fault + batch + 诊断摘要 |
| `make all` | functional + fault-all + 清理中间文件 |
| `make clean` | 清理 sim/work 与覆盖率编译中间产物 |

## 5. 编译选项

```
-full64 -sverilog -debug_access+all -kdb +define+FSDB -timescale=1ns/1ps
```

注错全量 bit 扫描额外定义：`+define+FI_ARRAY_BIT_TARGETS`

## 6. 波形查看（可选，需保留 sim/work 时）

默认运行结束后会删除 `sim/work/`（含波形 FSDB）。若需波形，在 `make finalize` 前手动查看，或单独运行 `make full` 后不执行 `make clean-work`。

```bash
make verdi-full    # 需 sim/work/waves/full.fsdb 存在
make verdi-fault
```

## 7. 诊断覆盖率说明

按赛题要求，注错结果分类为：已纠正、已探知、未探知。

保护概率 = (错误已纠正数量 + 错误已探知数量) / 错误总数

仿真结束后查看 `sim/fault_injection/diagnostic_coverage_summary.txt` 及 `reports/` 下 CSV。

## 8. 代码覆盖率说明

覆盖率数据收集于 `sim/functional/coverage/build.vdb`。
若 `urg` 生成 HTML 失败，可手动执行:

```bash
cd submission/scripts
urg -full64 -dir ../sim/functional/coverage/build.vdb -report ../sim/functional/coverage/urgReport
```

或重新运行 `make cov-report`（Makefile 已配置 urg 失败时不中断流程）。
