2. 模块功能点描述
2.1 基础设计规范
	模块为同步设计，时钟上升沿采样。
	同步复位设计。
	以 AXI4 为基础协议规范。
	AXI Safety Island 为设计组件，只能使用 RTL 进行设计。
2.2 接口定义
	1 路 AXI Slave 接口 (配置接口):
	64bits 数据位宽。
	32bits 地址位宽。
	用于接收上层配置信息（读取间隔、基地址、偏移量、Mask 等）。
	5 路 AXI Master 接口 (监控接口):
	64bits 数据位宽。
	32bits 地址位宽。
	按 AXI 总线协议进行读取，需配有 Timeout 机制（超时未返回则判定为故障）。
	AXI协议支持：
	需支持AXI基本读写操作
	需支持AXI outstanding操作
	可选支持out_of_order（附加分）
	可选支持interleaving 操作（附加分）
	需支持 OK/Error 类型response
	需支持 wrapper/incr burst类型
	需支持不小于16 length burst操作
	2/3路故障检测信号输出:
	fault_detect 信号，高电平有效，表示监控到系统fault发生。
	safety_island_fault_detect 信号，高电平有效，表示safety island自身fault发生。
	可选：safety_island_latent_fault_detect信号，高电平有效，表示safety island的潜在故障发生。(附加分)
2.3 核心功能逻辑
	周期读取任务: 安全岛的主要任务是按一定的时间间隔对各个安全寄存器进行读取。
	数据处理: 将读回来的值进行按位或 (Bitwise OR) 操作。
	故障判定: 根据按位或后的结果，给出 fault_detect 信号。
	配置参数 (通过 Slave 接口配置):
	读取间隔时钟周期 (Read Interval)。
	5 个读取基地址 (Base Address)，对应5路AXI Master。
	每个基地址对应 64 个偏移地址 (Offset Address)。
	每个地址的 Burst Type 和 Length。
	每个地址的 Mask 值 (用于屏蔽无效位)。
2.4 验证组件说明 (Testbench)
	AXI Slave 接口驱动模块: 由 Testbench 模拟master端，负责发送配置指令并反馈响应。
	AXI Master 接口驱动模块: 由 Testbench 模拟 Slave 端，负责响应读取请求，可模拟正常数据、超时、错误响应等场景。
	验证语言：可以使用 SV/UVM 等验证语言和方法学编写，不得使用第三方VIP。
3. 模块功能安全需求
本模块需满足 ASIL-D 等级的功能安全要求，针对单点故障和潜在故障（可选，附加分）进行防护。
3.1 永久故障 (Permanent Fault)
当模块内部出现一个点的永久故障（随机一个点出现 stuck at 0/1）：
	要求： 不影响模块的正常功能 或者 模块的检错信号在错误发生后的 10 个 cycle 内 或者 错误传递到 output port 时将safety_island_fault_detect置位报错。
	故障范围： 故障可能出现在 memory/combi logic/port/register 等随机电路位置。
	故障特性： 故障可以出现在正常工作时的任意时刻，并永久持续。
3.2 瞬时故障 (Transient Fault)
当模块内部出现一个点的瞬时故障（随机一个点出现持续时间大于一个 cycle，小于两个 cycle 的翻转）：
	要求： 不影响模块的正常功能 或者 模块的检错信号在错误发生后的 10 个 cycle 内 或者 错误传递到 output port 时将safety_island_fault_detect置位报错。
	故障范围： 故障可能出现在 memory/register 等随机电路位置。
3.3 外部总线故障防护
	当 AXI Master 读取外部寄存器发生超时 (Timeout) 或返回 Error Response 时，模块应能正确识别并置位 fault_detect 信号。
	配置寄存器本身需具备写保护或校验机制，防止非法配置导致安全功能失效，如有非法配置情况出现，置位fault_detect 信号。
3.4 允许AoU要求
	允许模块附带AoU，要求对接端配合模块进行安全校验等行为。例如：AoU_001: 总线信号校验位逻辑匹配。要求对接模块具有校验位信号产生/校验功能。
	如果设置AoU要求，则需要设置对应的错误探测机制。如当 AXI Master 读取外部寄存器时发生校验位错误，需置位fault_detect 信号。
4. 测试平台及用例要求
4.1 设计完善要求
	赛题提供基本设计规范文档。
	参赛者需要对基本设计规范文档进行完善，明确、详细说明设计所支持的功能点（功能点支持完善与否会纳入设计评分范围内）。
	参赛者完成对设计的 RTL 编码和功能验证，提交设计、验证等工程文件，并附加 ReadMe 文件说明对功能验证仿真进行复现的详细步骤。
4.2 故障注入要求
	参赛者需要建立测试平台的注错机制，对功能安全需求中描述的错误进行注入仿真。
	注错机制需覆盖：
	内部寄存器/组合逻辑单点故障 (Stuck-at-0/1)。
	内部寄存器瞬时翻转故障。
	AXI 总线传输超时/错误响应模拟。
4.3 结果检查与统计
	参赛者需要建立注错仿真的结果检查机制，并按功能安全需求的要求对注错仿真结果进行分类（错误已纠正、错误已探知、错误未探知…）。
	对不同类型的错误注入结果进行统计，并给出错误保护列表和概率： "保护概率"=("错误已纠正数量" +"错误已探知数量" )/"错误总数" 
	需证明设计满足 ASIL-D 对单点故障度量 (SPFM) 和潜在故障度量 (LFM) 的要求（建议参考ISO26262标准）。

一、 基本功能测试点

设计规范中明确要求的feature，必须包含在测试用例中。

1. 接口与协议测试

AXI Slave 配置接口测试：
验证配置参数的写入：读取间隔、基地址、偏移地址、Mask 值。
验证 64bit 数据位宽和 32bit 地址位宽的正确性。
AXI Master 监控接口测试 (5路)：
验证 Master 接口能正确发起读取请求。
超时机制测试： 模拟 Slave 端超时未响应，验证模块能否在超时后正确判定故障并置位 fault_detect。
AXI 协议基础支持测试：
验证基本读写操作。
验证 OK/Error 类型的 Response 处理。
验证 Wrapper/Incremental Burst 类型的支持。
验证模块支持 AXI outstanding 操作。
验证不小于 16 length 的 Burst 操作。
2. 核心逻辑功能测试

周期读取任务测试：
验证模块能按照配置的时钟周期间隔对安全寄存器进行读取。
数据处理逻辑测试：
验证读回的数据能正确进行按位或 操作。
故障判定逻辑测试：
验证当检测到系统故障时，fault_detect 信号能被正确置位。
3. 功能安全需求测试

永久故障 (Stuck-at) 注入测试：
在 Memory、Combi Logic、Port、Register 等位置注入 Stuck-at-0 和 Stuck-at-1。
验证故障不影响正常功能，或者在错误发生后 10 个 cycle 内置位 safety_island_fault_dete。
瞬时故障测试：
在 Register 等位置注入单周期翻转故障。
验证模块能否在错误发生后 10 个 cycle 内或传递到 output port 时置位 safety_island_fault_dete。
外部总线故障测试：
模拟 AXI 总线传输超时。
模拟 AXI 总线返回 Error Response。
验证模块能否正确识别并置位 fault_detect。
如有AoU要求外部模块进行校验码传递，则需要模拟校验码错误情况，验证模块能否正确识别并置位fault_detect。
配置寄存器保护测试：
验证配置寄存器是否具备写保护或校验机制，防止非法配置导致安全功能失效。如有非法配置，模块能否置位fault_detect。
4. 验证平台测试

仿真复现测试：
提交 ReadMe 文件，详细描述赛题项目目录结构，测试用例列表，以及验证测试用例的复现步骤。
提交运行脚本文件，能复现VCS仿真验证的行为。如环境对工具版本有具体要求，提供版本设置脚本，或者在ReadMe文件中说明。
可选ZOIX工具进行错误仿真，需提供ZOIX仿真运行环境和脚本（附加分）。
