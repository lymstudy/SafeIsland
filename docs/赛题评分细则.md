# 赛题评分细则（清晰版）

## 输入与输出（简述）

- 输入：赛题提供的安全岛设计规范与基础测试要求。
- 输出（参赛者需提交）：
  - 设计文档（含失效模型与安全机制设计）。
  - RTL 源码与功能仿真（含 testbench 与复现脚本）。
  - 注错仿真环境、用例与结果（诊断覆盖率统计）。
  - 工程说明与自动化运行脚本（需包含 VCS 自动执行脚本）。

## 评分结构（总分 100）

1) 模块基础功能实现 — 30 分
	- 设计文档补充（10 分）: 实现思路、支持 feature 列表。
	- RTL + 功能验证（20 分）: RTL 完成且 testbench 验证通过。

2) 安全性理论分析与文档 — 20 分
	- Memory/寄存器失效分析（10 分）: 列出失效点、后果、保护手段与效果。
	- 数字逻辑失效分析（10 分）: 同上。

3) 安全机制实现 — 30 分
	- Memory/寄存器保护（10 分）: 探测型最高得 60%，纠错型最高得 100%。
	- 数字逻辑保护（20 分）: 探测型最高得 60%，纠错型最高得 100%。

	注：纠错型（如 ECC、TMR）得分高于仅探测型（如奇偶校验、互锁）。

4) 注错测试与覆盖 — 20 分
	- Memory/寄存器注错用例覆盖（10 分）: 按 bits 覆盖率计分（示例：256 bits 总量，覆盖 128 bits → 50%）。
	- 数字逻辑注错覆盖（10 分）: 按覆盖比例计分。

### 附加分与平分规则

- 若出现队伍总分平分，则根据设计文档中的附加功能点（实现数量与质量）进行排名决胜。

## 提交清单（必须包含）

- 完整设计文档、失效模型与安全机制设计说明。
- RTL 源码与功能仿真环境（含 testbench、VCS 自动化脚本）。
- 注错仿真计划、用例、运行说明与结果（含诊断覆盖率统计）。
- 功能仿真结果（含代码行覆盖率）。

## 评分重点速览

- 功能实现与可复现实验（30 分）
- 安全性分析完整性（20 分）
- 纠错类实现质量（30 分）
- 注错覆盖与验证报告（20 分）
