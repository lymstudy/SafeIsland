## 项目四步计划 — 最终状态

### 第 1 步：功能验证 ✅

ModelSim Full TB 34/34 PASS，覆盖：
- AXI Slave 配置读写、5路Master、outstanding、out-of-order、interleaving
- CRC-8/16 E2E、周期扫描、Mask+OR、fault_detect、超时、Error Response
- 心跳自检、KAT、TMR、Write-Verify、Expected Compare

### 第 2 步：失效模型 ✅

`docs/失效模型分析.md`：
- Memory/寄存器：~117K bits 逐模块分析（含所有 shadow/inv/TMR/CRC 保护）
- 数字逻辑：65 个 fault site，66% 有保护，34% 无保护（深组合逻辑，需 formal/网表级注错或等效论证）

### 第 3 步：保护机制 ✅

| 保护机制 | 对象 | 类型 |
|---------|------|:--:|
| Shadow/反码 | 全量配置寄存器 + top rsp FIFO + read_engine slot + fault_detector + heartbeat | 探测 |
| TMR | FSM state、fault 输出、cfg_locked、cfg_illegal、enable、safety_fault_q、error_code | 纠错 |
| CRC-16 E2E | AR 通道签名 + R 通道验证 | 探测 |
| 心跳自检 | fault_detect 输出路径 | 探测 |
| KAT | 完整读路径预验证 | 纠错 |
| Write-Verify | S_AXI 配置写入 | 探测 |
| 超时/非法配置检测 | AXI 总线 + 配置参数 | 探测 |

### 第 4 步：注错平台 ✅

- `tools/run_fault_batch.py` — 一键批量注错：编译 → 运行 → CSV → 覆盖率报告
- FI TB 支持三种模式：基线（38 case）、全 bit 扫描（594 case）、单点注入（+FAULT_KIND +BIT_INDEX）
- 数字 logic fault site 覆盖 24/65 (37%)，剩余为深组合逻辑/原子函数，force 方案物理上限
