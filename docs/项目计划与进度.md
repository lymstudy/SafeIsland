## 项目四步计划 — 最终状态

### 第 1 步：功能验证 ✅

**ModelSim 34/34 PASS + VCS 34/34 PASS。** 两份仿真器互验通过。

覆盖：
- AXI Slave 配置读写、5路Master、outstanding、out-of-order、interleaving
- CRC-16 E2E、周期扫描、Mask+OR、fault_detect、超时、Error Response
- 心跳自检、KAT、TMR、Write-Verify、Expected Compare、非法配置检测

### 第 2 步：失效模型 ✅

`docs/失效模型分析.md` — 逐模块完整分析：
- Memory/寄存器：~117K bits，95%+ 有保护（shadow/inv + TMR）
- 数字逻辑：65 fault sites，66% 有保护，34% 为深组合逻辑

### 第 3 步：保护机制 ✅

| 类型 | 机制 | 对象 |
|:--:|------|------|
| 纠错 | TMR | FSM state、fault 输出、cfg_locked、cfg_illegal、enable、safety_fault_q、error_code |
| 纠错 | KAT | 完整读路径 AR → Interconnect → Slave → R |
| 探测 | Shadow/反码 | 全部配置寄存器、top rsp FIFO、read_engine slot、fault_detector、heartbeat |
| 探测 | CRC-16 E2E | AR 通道签名 + R 通道双路计算比对 |
| 探测 | 心跳自检 | fault_detect 输出路径周期性验证 |
| 探测 | Write-Verify | S_AXI 配置写入一致性 |
| 探测 | 超时/非法配置 | AXI 总线超时 + 配置参数合法性 |

### 第 4 步：注错平台 ✅

| 能力 | 工具 |
|------|------|
| 基线 38 case | `make fault` (VCS) / `run_safety_island_fault_injection_tb.do` (ModelSim) |
| 全量 bit 扫描 594 case | `make batch` (VCS) / `run_fault_batch.py` (ModelSim) |
| 单点注入 | `+FAULT_KIND=<name> +BIT_INDEX=<n>` |
| 覆盖率报告 | `python tools/run_fault_batch.py` → CSV + 终端报告 |

```powershell
& 'D:\software\modelism\MODELISM\win64\vsim.exe' -c -do run_safety_island_top_full_tb.do

```
