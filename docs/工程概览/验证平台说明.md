## 3. 全功能测试场景 (34 用例)

### 3.1 必选功能 (15 场景)

| # | 场景名称 | 覆盖要点 |
|---|---------|---------|
| 1 | `basic_fault_flow` | 外部寄存器 mismatch → fault_detect |
| 2 | `no_fault_flow` | 正常扫描，无误报 |
| 3 | `multi_master_flow` | 多路 Master + 多 entry + Mask+OR |
| 4 | `burst_16_incr_flow` | 16 beat INCR burst OR 汇总 |
| 5 | `wrap_burst_flow` | 合法 WRAP burst |
| 6 | `bus_error_flow` | RRESP=SLVERR → fault_detect |
| 7 | `timeout_flow` | R 通道不返回 → timeout 故障 |
| 8 | `outstanding_flow` | 同路 Master 4 outstanding 顺序返回 |
| 9 | `invalid_rid_error_flow` | 未分配 RID → bus fault |
| 10 | `aou_rcheck_error_flow` | CRC-8 单 beat 校验错误 |
| 11 | `aou_rcheck_burst_error_flow` | CRC-8 多 beat 中间校检错误 |
| 12 | `config_error_flow` | interval=0/非法 burst/非法 WRAP/lock 后写 |
| 13 | `expected_mismatch_flow` | data ≠ expected → fault |
| 14 | `expected_match_flow` | data == expected 无误报 |
| 15 | `expected_masked_match_flow` | 部分 mask 比较正确 |

### 3.2 可选功能 (7 场景)

| # | 场景名称 | 覆盖要点 |
|---|---------|---------|
| 16 | `out_of_order_flow` | 4 outstanding 乱序返回 (3→2→1→0) |
| 17 | `interleaving_flow` | 多 burst R beat 按 RID 交错 |
| 18 | `out_of_order_interleaving_flow` | 乱序 + 交织组合 |
| 19 | `aou_rcheck_ok_flow` | CRC-8 正确 → 功能正常 |
| 20 | `latent_fault_flow` | 配置 shadow/反码注错 → latent fault |
| 21 | `wrap_burst_16_flow` | 16 beat WRAP burst |
| 22 | `write_verify_fault_flow` | S_AXI 写后 Shadow 错误 |

### 3.3 ASIL-D 增强 (12 场景，含 CRC-16)

| # | 场景名称 | 覆盖要点 |
|---|---------|---------|
| 23-28 | `*_crc16_*` | CRC-16 E2E 模式各种场景 |
| 29 | `kat_ok_flow` | KAT 已知寄存器验证通过 |
| 30 | `kat_mismatch_flow` | KAT 预期值不匹配 → fault |
| 31 | `heartbeat_ok_flow` | 心跳自检正常通过 |
| 32 | `heartbeat_inject_flow` | 心跳注入 → 检测路径验证 |
| 33 | `write_verify_ok_flow` | Write-Verify 正常 |
| 34 | `tmr_recovery_flow` | TMR 单份故障 → 多数表决恢复 |

## 5. 运行方法

### 5.1 ModelSim (Windows)

```powershell
# 全功能测试
cd sim/modelsim
& 'D:\software\modelism\MODELISM\win64\vsim.exe' -c -do run_safety_island_top_full_tb.do

# 故障注入
& 'D:\software\modelism\MODELISM\win64\vsim.exe' -c -do run_safety_island_fault_injection_tb.do

# 查看波形
& 'D:\software\modelism\MODELISM\win64\vsim.exe' -view sim/out/safety_island_top_full/safety_island_top_full.wlf
```

### 5.2 VCS (Linux)

```bash
cd sim/vcs

# 全功能回归
make full

# 故障注入
make fault

# 批量 bit sweep
make batch

# 故障检测单元测试
make fdet

# 全部回归
make regress

# Verdi 波形
make verdi-full
```

### 5.3 Icarus Verilog

```bash
# 模块级测试
python tools/run_tests.py --level module

# 顶层测试
python tools/run_tests.py --level top

# 全部
python tools/run_tests.py --level all
```

### 5.4 Python 批量故障注入

```bash
# 生成 bit 清单
python tools/gen_fi_bit_list.py --full
python tools/gen_fi_bit_list.py --output fault_campaign/smoke_list.csv

# 批量运行
python tools/run_fi_sweep.py --fault-list fault_campaign/fault_list.csv

# 分析结果
python tools/analyze_fi_report.py --input sim/out/safety_island_fault_sweep/fault_sweep_report.csv

# 生成安全报告
python tools/gen_safety_report.py --input fault_campaign/fault_list.csv
```

## 6. 结果检查机制

### 6.1 全功能 TB 判定

```
PASS: <testcase_name> — 场景通过
FAIL: <testcase_name> — 场景失败
最终输出: cases=N, Errors: X
```

### 6.2 故障注入 TB 判定

每个故障注入 case 输出 CSV 行：

| 字段 | 说明 |
|------|------|
| name | 故障名称 |
| type | 故障分类 (detected/corrected/undetected) |
| result | 注入结果 |
| cycles | 检测延迟 (cycle 数) |
| error_code | 对应错误码 |
| fault_detect | 系统故障是否置位 |
| safety_fault | 安全岛故障是否置位 |
| latent_fault | 潜在故障是否置位 |

### 6.3 汇总统计

```
FI_SUMMARY: total=N corrected=C detected=D undetected=U protection_rate=P%
```
