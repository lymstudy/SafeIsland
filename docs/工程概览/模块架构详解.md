# AXI Safety Island — 模块架构详解

> 详细描述各 RTL 模块的内部设计、接口信号与安全机制

---

## 1. safety_island_top — 顶层集成

**文件**: `rtl/safety_island_top.v` | **行数**: ~670

### 1.1 功能说明

顶层模块实例化所有子模块并完成信号连接：

- **safety_island_axi_config_slave** — 配置接口
- **safety_island_core_logic** — 核心控制
- **safety_island_fault_detector** — 故障检测
- **safety_island_heartbeat** — 心跳自检
- **safety_island_axi_read_engine ×5** (generate) — 读引擎

### 1.2 RSP FIFO (generate 块内)

每个 Master 通道内置响应缓冲 FIFO：
- 读引擎输出为单周期脉冲 (`done/error/timeout`)，需缓冲为持久 valid
- 逆序/反码指针和存储提供 Shadow 校验
- `rsp_fifo_shadow_error_comb` 综合 OR 所有逆序失配
- `rsp_fifo_safety_fault[mi]` 检测 shadow 错误 + 指针/计数越界

### 1.3 顶层 TMR 故障输出

```verilog
// 故障检测 TMR (三份复制 + 多数表决)
wire fd_a = fd_external_fault | fd_bus_fault | fd_cfg_fault;  // DONT_TOUCH
wire fd_b = fd_external_fault | fd_bus_fault | fd_cfg_fault;  // DONT_TOUCH
wire fd_c = fd_external_fault | fd_bus_fault | fd_cfg_fault;  // DONT_TOUCH

assign fault_detect = (fd_a & fd_b) | (fd_b & fd_c) | (fd_a & fd_c);
assign fd_tmr_mismatch = (fd_a ^ fd_b) | (fd_a ^ fd_c) | (fd_b ^ fd_c);

// 安全岛故障检测 TMR
assign safety_island_fault_detect =
    ((sifd_a & sifd_b) | (sifd_b & sifd_c) | (sifd_a & sifd_c))
    | fd_tmr_mismatch | sifd_tmr_mismatch;
```

### 1.4 接口信号

| 接口 | 方向 | 位宽 | 说明 |
|------|:--:|------|------|
| S_AXI | Slave | 64b data / 32b addr | 配置接口 |
| M_AXI ×5 | Master | 64b data / 32b addr | 监控读接口 (flat 展平) |
| fault_detect | Output | 1b | 系统故障检测 (TMR) |
| safety_island_fault_detect | Output | 1b | 安全岛自身故障 (TMR) |
| safety_island_latent_fault_detect | Output | 1b | 潜在故障 |
| fault_or_result | Output | 64b | Mask+OR 累加结果 |
| core_error_code | Output | 8b | 核心错误码 |

## 2. safety_island_axi_config_slave — AXI 配置 Slave

**文件**: `rtl/safety_island_axi_config_slave.v`

### 2.1 功能说明

- 实现 AXI4-Lite Slave 协议 (读/写通道)
- 管理所有配置寄存器 (控制/间隔/基地址/偏移/Mask/Burst/Expected/KAT)
- 配置锁定机制：lock 后写返回 SLVERR 并触发配置故障
- 写保护：配置期间 scan_busy 返回 SLVERR
- **Write-Verify**：写后 Shadow 一致性检查

### 2.2 寄存器保护

| 寄存器组 | 保护方式 | 说明 |
|---------|---------|------|
| enable | TMR (3 份) + Shadow + 反码 | 3×1bit |
| cfg_locked_r | TMR (3 份) | 配置锁状态 |
| cfg_illegal_r | TMR (3 份) | 非法配置标志 |
| read_interval | Shadow 反码 | 2×64bit |
| base_addr_q[0:4] | Shadow 反码 | 2×5×32bit = 320bit |
| offset_q[0:319] | Shadow 反码 | 2×320×32bit = 20480bit |
| mask_q[0:319] | Shadow 反码 | 2×320×64bit = 40960bit |
| burst_type_q[0:319] | Shadow 反码 | 2×320×2bit = 1280bit |
| burst_len_q[0:319] | Shadow 反码 | 2×320×8bit = 5120bit |
| expected_q[0:319] | Shadow 反码 | 2×320×64bit = 40960bit |
| entry_valid_q[0:319] | Shadow 反码 | 2×320×1bit = 640bit |
| KAT 寄存器 | Shadow 反码 | kat_enable/addr/expected/mask |
| AXI 通道暂存 | 无 | ~210bit，AXI 协议暂存 |

### 2.3 地址空间布局

```
0x0000_0000  CONTROL         控制/状态
0x0000_0008  READ_INTERVAL   扫描间隔
0x0000_0010  STATUS          状态
0x0000_0018  FAULT_RESULT    OR 结果
0x0000_0020  ERROR_CODE      错误码
0x0000_0028  INDEX_STATUS    当前索引
0x0000_0030  OUTSTANDING     Outstanding 计数
0x0000_0038  KAT_CTRL        KAT 控制
0x0000_0040  KAT_ADDR        KAT 地址
0x0000_0048  KAT_EXPECTED    KAT 预期值
0x0000_0050  KAT_MASK        KAT 掩码
0x0000_0100  BASE[0..4]      5路基地址
0x0000_1000  ENTRY[0..4][0..63]  5×64 条目配置区
    每个 ENTRY: +0x00 offset, +0x08 mask, +0x10 burst, +0x18 expected
```

## 3. safety_island_core_logic — 核心控制逻辑

**文件**: `rtl/safety_island_core_logic.v` | **行数**: ~800+

### 3.1 主要功能

1. **扫描调度 FSM** — 周期/单次扫描控制
2. **地址生成** — `current_read_addr = base_addr[mi] + offset[mi][ei]`
3. **Outstanding 管理** — Pending FIFO 缓冲未完成请求
4. **数据路由** — 读响应从 read_engine 路由到 fault_detector
5. **KAT (Known-Answer Test)** — 预扫描已知寄存器验证完整读路径
6. **安全自检** — 20+ 项组合逻辑实时检查

### 3.2 FSM 设计

```
IDLE → WAIT_INTERVAL → PREP_SCAN → ISSUE_REQ → DRAIN → IDLE
                                    ↑            ↓
                                    └─── WAIT ───┘
SAFE_ERROR (从任意状态进入)
```

- **TMR 保护**：`state_a/b/c` 三份 + `state_inv` 反码
- **非法状态检测**：11 个合法状态，其他 → ST_SAFE_ERROR

### 3.3 Pending FIFO

- 深度 = `MAX_OUTSTANDING` (默认 4)
- 每个 entry 存储：mask, expected, master_idx, entry_idx, valid
- 保护：写指针/读指针范围检查, valid 标志交叉检查

### 3.4 安全自检清单

| 检查项 | 信号 | 检测方式 |
|-------|------|---------|
| FSM 状态非法 | `fsm_state_illegal_comb` | 组合比较 11 个合法编码 |
| FSM 反码失配 | `fsm_state_inv_comb` | state ≠ ~state_inv |
| 累加器反码失配 | `accum_shadow_fault_comb` | accum ≠ ~accum_inv |
| 当前索引越界 | `current_index_fault_comb` | mi ≥ NUM_MASTERS, ei ≥ NUM_ENTRIES |
| Pending 索引越界 | `pending_index_fault_comb` | 同上 |
| Pending 指针越界 | `pending_ptr_fault_comb` | ptr ≥ MAX_OUTSTANDING |
| Pending valid 异常 | `pending_valid_fault_comb` | FIFO 空时 pop 或满时 push |
| Outstanding 溢出 | `outstanding_fault_comb` | count > MAX_OUTSTANDING |
| KAT 比较失败 | `kat_fail_comb` | rd_data & mask ≠ expected & mask |
| Burst 合法性 | `cfg_fault_comb` | burst type/length 检查 |

## 4. safety_island_axi_read_engine — AXI 读引擎

**文件**: `rtl/safety_island_axi_read_engine.v`

### 4.1 功能说明

- **只读 Master**：仅实现 AR + R 通道 (AW/W/B 固定 tie-off)
- **Slot 管理**：最多 `MAX_OUTSTANDING` (4) 个未完成 AR
- **Out-of-Order** 支持：按 slot (ARID = MASTER_ID ^ slot_index) 独立管理
- **Interleaving** 支持：每个 slot 独立维护 beat count 和累加器
- **CRC E2E 校验** (`CRC_WIDTH` = 8 或 16)

### 4.2 CRC E2E 校验

#### CRC-8 模式 (CRC_WIDTH=8)
- 校验对象：`{RID, RDATA, RRESP, RLAST}`
- 多项式：`0x07` (x^8 + x^2 + x + 1)
- 初始值：`0x00`

#### CRC-16 模式 (CRC_WIDTH=16)
- **两阶段** E2E 签名：
- **AR 阶段**：`ar_signature = CRC16({ARID, ARADDR, ARLEN, ARSIZE, ARBURST})`
- **R 阶段**：`r_crc_expected = CRC16({ar_signature, RID, RDATA, RRESP, RLAST})`
- 多项式：`0x1021` (x^16 + x^12 + x^5 + 1)
- 初始值：`0xFFFF`
- **双路计算**：`r_crc_expected` 和 `r_crc_expected_dup` 独立计算比对

### 4.3 Slot 结构

每个 Slot 维护：
- `slot_id_q` — ARID
- `slot_len_q` — 突发长度
- `slot_beat_q` — 当前 beat 计数
- `slot_age_q` — 超时计时器
- `slot_accum_q` — 读数据 OR 累加
- `slot_ar_sig_q` — AR 通道签名
- `slot_error_q` / `slot_timeout_q` — 异常标志
- **全量 Shadow/反码保护**

### 4.4 Timeout 机制

- 每 slot 独立 `slot_age` 计数
- 每次 AR fire 启动计时
- 每次 R beat 重置计时
- 超过 `TIMEOUT_CYCLES-1` → timeout
- 超时后 `slot_valid` 保持直到 core logic 消费

## 5. safety_island_fault_detector — 故障检测器

**文件**: `rtl/safety_island_fault_detector.v`

### 5.1 功能说明

| 检测类型 | 实现方式 | 输出 |
|---------|---------|------|
| **Mask+OR** | 逐响应 `accum \|= data & mask` | `fault_or_result` |
| **Expected 比较** | `(accum & mask) != (expected & mask)` | `external_fault_event` |
| **Stuck-at** | 连续 N 轮 `ch_mismatch_this_round` | `safety_island_fault_event` |
| **Latent** | 曾 mismatch 但自行恢复 → 跨轮 latch | `safety_island_latent_fault_event` |
| **Bus fault** | AXI 响应错误/超时 透传 | `bus_fault_event` |
| **Config fault** | 非法配置/Shadow 错误 透传 | `cfg_fault_event` |
| **Internal fault** | Core logic 安全自检故障 透传 | `safety_island_fault_event` |

### 5.2 Stuck-at 检测

```
if (ch_mismatch_this_round[ch]) begin
    ch_stuck_counter[ch] <= ch_stuck_counter[ch] + 1;
end else begin
    ch_stuck_counter[ch] <= 0;
end

if (ch_stuck_counter[ch] >= STUCK_AT_THRESHOLD)
    safety_island_fault_event <= 1;
```

### 5.3 Latent 故障检测

```
if (ch_mismatch_this_round[ch])
    ch_mismatch_latched[ch] <= 1;  // latch

if (scan_done_pulse) begin
    if (ch_mismatch_latched[ch] && !ch_mismatch_this_round[ch])
        latent_fault_event <= 1;   // 曾发生但已恢复
    ch_mismatch_latched[ch] <= 0;  // 每轮清
end
```

## 6. safety_island_heartbeat — 心跳自检

**文件**: `rtl/safety_island_heartbeat.v` | **参数**: `HEARTBEAT_INTERVAL=1024`

### 6.1 工作原理

```
H_IDLE → H_WAIT_IDLE → H_INJECT → H_WAIT_DET → H_CLEAR → H_IDLE
                                                  ↓
                                               H_FAIL (自锁)
```

1. 每 1024 周期，等待 scan_busy 空闲
2. 注入测试故障 (`test_inject = 1`)
3. 等待 10 个 cycle 内观察到 `safety_island_fault_detect` 置位
4. 如果 10 cycle 内未检测到 → `heartbeat_fault = 1` (自锁到 rst)
5. 清除注入，返回 IDLE

### 6.2 保护机制

| 信号 | 保护 |
|------|------|
| state | Shadow/反码 |
| counter | Shadow/反码 |
| wait_cycles | Shadow/反码 |
| heartbeat_fault | 自锁到 rst |
| heartbeat_internal_fault | 3 路 shadow 比较 |

## 7. tmr_voter — TMR 表决器

**文件**: `rtl/tmr_voter.v`

### 7.1 功能

纯组合逻辑，2-out-of-3 多数表决：

```verilog
assign voted    = (a & b) | (b & c) | (a & c);
assign mismatch = |((a ^ b) | (a ^ c) | (b ^ c));
```

### 7.2 使用位置

- `safety_island_top.v`：故障输出 TMR (fd_a/b/c, sifd_a/b/c)
- `safety_island_core_logic.v`：FSM state_a/b/c, safety_fault_q, safety_error_code_q
- `safety_island_axi_config_slave.v`：cfg_locked_r, cfg_illegal_r, enable

## 8. 兼容版/融合版差异

| 维度 | 整合版 (safety_island_top) | 融合版 (axi_safety_island_core) |
|------|--------------------------|-------------------------------|
| 模块数 | 5 核心 + 心跳 + TMR | 9 模块 (细粒度拆分) |
| 文件列表 | `safety_island_top.f` | `axi_safety_island_pkg.vh` + 各模块 |
| 故障检测 | 独立模块 `safety_island_fault_detector` | `fault_detector` + `fault_status_manager` |
| 调度器 | 集成在 core_logic | 独立 `monitor_scheduler` |
| Expected 支持 | ✅ | 通过 `read_data_processor` |
| 验证状态 | ModelSim VCS 双平台全通 | 部分模块级测试有失败 |
| 推荐使用 | ✅ **推荐** | 备用/参考 |
