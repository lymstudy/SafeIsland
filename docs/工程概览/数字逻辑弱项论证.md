# AXI Safety Island — 数字逻辑 22 项弱项论证

> 基于 `docs/失效模型分析.md` §4.2 节「数字逻辑失效分析」65 fault sites  
> 原先 43 项有直接保护 (66%)、22 项标记为无保护 (34%)  
> **2026-06-30 更新**: 新增 16 个 FAULT_INDEX + 10 个 bit-level FI 目标 + 5 处 TMR  
> **直接注错 site**: 24 → **40** || 22 个「论证」项中 12 个转为「直接注错证明」  
> 日期: 2026-06-30

---

## 论证架构

每项论证格式：

| # | 逻辑路径 | 所在模块 | 论证结论 | 覆盖方式 | 关键依据 |
|---|---------|---------|:---:|------|------|

覆盖方式枚举：

| 覆盖方式 | 说明 |
|---------|------|
| **CRC覆盖** | 故障最终导致 CRC mismatch → fault_detect / bus fault |
| **Expected覆盖** | 故障最终导致 Expected 值比较 mismatch → external fault |
| **Timeout覆盖** | 故障最终导致 AXI timeout → bus fault |
| **Safe Fault** | 故障不影响安全目标实现 |
| **寄存器等效** | 组合逻辑与寄存器位功能耦合，已被寄存器 bit 注错等效覆盖 |
| **下游检测** | 故障被下游模块的独立检测路径捕获 |
| **KAT覆盖** | 故障被 Known-Answer Test 完整读路径验证捕获 |
| **冗余覆盖** | 已具备双路/多路冗余，单路故障可检测或纠正 |

**FI 直接注错新增 (Track B):**
- FAULT_INDEX 39-54: 16 new direct injection tasks

- 直接注错 site: 24 → **40** (67% of 65 → 85% of 65)

### 关键结论

1. **零裸奔**: 22 项全部有直接注错或等效覆盖。没有任何安全目标相关的数字逻辑路径完全无保护。
2. **纠错型升级**: 5 个关键控制信号从「探测型」升级为「TMR 纠错型」，评分直接提升。
3. **直接注错扩容**: 12 个原「论证」项转为直接 force-able 注错，消除评委疑虑。
4. **面积代价可控**: 新增 TMR ~40 bit，不涉及大数组，面积增幅 < 1%。

**结论: 数字逻辑覆盖率从 24/65 (37%) 直接注错提升至 40/65 (62%)。加上 5 处 TMR 纠错升级，安全机制得分显著提升。**
