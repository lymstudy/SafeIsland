## 2. run_tests.py — Icarus 测试运行器

### 用法

```bash
python tools/run_tests.py --level module     # 模块级测试 (4 个)
python tools/run_tests.py --level top         # 顶层集成测试 (1 个)
python tools/run_tests.py --level fault       # 故障注入测试 (1 个)
python tools/run_tests.py --level all         # 全部运行
```

### 测试套件配置

```python
TEST_SUITES = {
    "module": [
        {"name": "s_axi_config",        "rtl": ["s_axi_config.v"],                 "tb": "tb_s_axi_config.v"},
        {"name": "config_checker",      "rtl": ["config_checker.v"],               "tb": "tb_config_checker.v"},
        {"name": "axi_master_channel",  "rtl": ["axi_master_channel.v"],           "tb": "tb_axi_master_channel.v"},
        {"name": "data_fault",          "rtl": ["read_data_processor.v", ...],     "tb": "tb_data_fault.v"},
    ],
    "top": [
        {"name": "top_integration",     "rtl": [所有 rtl/*.v],                     "tb": "tb_top.v"},
    ],
    "fault": [
        {"name": "fault_campaign",      "rtl": [所有 rtl/*.v],                     "tb": "tb_fault_campaign.v"},
    ],
}
```

### 运行流程

1. 使用 `iverilog -gsystem-verilog` 编译 RTL + TB
2. 使用 `vvp` 执行仿真
3. 检查输出中 `PASS`/`FAIL` 关键字
4. 汇总结果

### 判定逻辑

| 输出特征 | 判定 |
|---------|:----:|
| `ALL TESTS PASSED` 或 `INTEGRATION TESTS PASSED` | PASS ✅ |
| `ERRORS` > 0 | FAIL ❌ |
| 其他 | UNKNOWN ⚠️ |

## 3. run_fi_sweep.py — ModelSim 批量故障注入

### 用法

```bash
# 默认运行 (使用 fault_campaign/fault_list.csv)
python tools/run_fi_sweep.py

# 指定故障列表和输出
python tools/run_fi_sweep.py \
    --fault-list fault_campaign/fault_bit_smoke_list.csv \
    --output sim/out/safety_island_fault_sweep/fault_sweep_report.csv

# 限制运行数量 (调试用)
python tools/run_fi_sweep.py --limit 5

# 静默模式 (不输出每个 case 的仿真日志)
python tools/run_fi_sweep.py --quiet
```

### 运行流程

```
1. 读取 CSV 故障清单 (fault_id, module, hierarchical_path, type, fault_kind, bit_index...)
2. 编译设计 (vlib → vmap → vlog RTL + vlog TB)
3. 遍历每行故障:
   a. 创建 case 独立目录
   b. 运行 vsim 传入 +FAULT_INDEX 或 +FAULT_KIND/+BIT_INDEX
   c. 收集 summary.txt 和 report.csv
4. 合并所有 case 的 CSV 为总报告
5. 输出统计: 总数/runner 失败数
```

### 故障清单 CSV 格式

| 字段 | 说明 | 示例 |
|------|------|------|
| `fault_id` | 故障唯一标识 | `B0001_032` |
| `module` | 所属模块 | `config_slave` |
| `hierarchical_path` | 层次化路径 | `dut.u_cfg.read_interval_inv[32]` |
| `type` | 故障类型 | `config_bit` / `core_bit` / `bus_fault` |
| `inject_cycle` | 注入时机 | `post_config` |
| `duration` | 持续时长 | `until_detected` |
| `target_ch` | 目标通道 | `0` |
| `expected_class` | 期望分类 | `detected` |
| `fault_kind` | 注入类别 | `cfg_read_interval_inv` / `core_state_inv` 等 |
| `bit_index` | 位索引 | `32` |

## 5. gen_safety_report.py — 安全指标报告

### 用法

```bash
python tools/gen_safety_report.py --input fault_campaign/fault_list.csv
```

### 输出内容

- **SPFM** (Single Point Fault Metric) = `1 - undetected_SPF / total_relevant`
- **LFM** (Latent Fault Metric) = `1 - latent / (total - SPF - RF)`
- 故障按模块/类型/分类分布
- ASIL-D 合规判定 (SPFM ≥ 99%, LFM ≥ 90%)

### 生成文件

- 控制台输出：完整安全指标报告
- CSV 导出：`fault_campaign/safety_report.csv`

## 6. analyze_fi_report.py — 故障注入结果分析

### 用法

```bash
# 分析 baseline 结果
python tools/analyze_fi_report.py \
    --input sim/out/safety_island_fault_injection/fault_injection_report.csv \
    --output fault_campaign/safety_report.csv

# 分析 bit sweep 结果
python tools/analyze_fi_report.py \
    --input sim/out/safety_island_fault_sweep/fault_sweep_report.csv \
    --output fault_campaign/safety_sweep_report.csv
```

### 输出指标

| 指标 | 说明 |
|------|------|
| Total Faults | 总故障数 |
| Detected | 已检测 |
| Corrected | 已纠正 |
| Safe / No effect | 安全/无影响 |
| Undetected | 未检测 |
| Campaign Diagnostic Coverage | 诊断覆盖率 = (passed / total) × 100% |

### 结果分类

```python
PASS_RESULTS = {"corrected", "detected", "safe"}
```
任何不在 `PASS_RESULTS` 中的结果视为 undetected。

## 7. VCS Makefile (sim/vcs/)

### 目标说明

| 目标 | 说明 |
|------|------|
| `make full` | 全功能 TB 编译+运行 |
| `make fault` | 故障注入 TB 编译+运行 |
| `make batch` | 批量 bit sweep (594 用例) 编译+运行 |
| `make fdet` | 故障检测器单元测试编译+运行 |
| `make regress` | 全部回归 (full + fault + fdet) |
| `make verdi-full` | 打开 Full TB 波形 |
| `make verdi-fault` | 打开 Fault Injection 波形 |
| `make verdi-fdet` | 打开 Fault Detector 波形 |
| `make clean` | 清理编译产物 |

### VCS 编译选项

| 选项 | 说明 |
|------|------|
| `-full64` | 64 位模式 |
| `-sverilog` | SystemVerilog 支持 |
| `-debug_access+all` | 全信号调试访问 |
| `-kdb` | Verdi KDB 数据库 |
| `+define+FSDB` | FSDB 波形生成 |
| `-timescale=1ns/1ps` | 仿真时间单位/精度 |
| `+define+FI_ARRAY_BIT_TARGETS` | 数组 bit 注入目标 (batch 模式) |

## 8. ModelSim 仿真脚本 (sim/modelsim/)

### 脚本清单

| 脚本 | 测试对象 |
|------|---------|
| `run_safety_island_top_basic_tb.do` | 基本功能测试 (~5 场景) |
| `run_safety_island_top_full_tb.do` | 全功能集成测试 (34 场景) |
| `run_safety_island_fault_injection_tb.do` | 故障注入测试 |
| `axi/run_axi_master_tb.do` | AXI Master 单元测试 |
| `axi/run_axi_slave_tb.do` | AXI Slave 单元测试 |

### .do 脚本结构 (以 full_tb 为例)

```tcl
# 1. 创建工作库
vlib work
vmap work work

# 2. 编译 RTL
vlog -work work -f ../../rtl/safety_island_top.f

# 3. 编译 TB
vlog -work work ../../tb/tb_safety_island_top_full.v

# 4. 运行仿真
vsim -c -voptargs=+acc work.tb_safety_island_top_full -do "run -all; quit -f"
```
