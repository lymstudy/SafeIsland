## 2. 核心概念速览

### 2.1 这个 IP 做什么？

```
配置 → 安全岛 → 周期性读取外部寄存器 → 故障检测 → 告警输出
```

1. 通过 **S_AXI 接口**配置：读哪些地址、多长时间读一次、用什么 Mask
2. 安全岛按照配置**周期性**通过 5 路 **M_AXI** 去读外部寄存器
3. 读回来的数据做：
   - **Mask+OR**：所有读回值按位 OR
   - **Expected 比较**：如果值不等于期望值 → 故障
   - **CRC 校验**：总线传输完整性检查
4. 如果发现异常，输出：
   - `fault_detect` — 系统有故障
   - `safety_island_fault_detect` — 安全岛自己有问题
   - `safety_island_latent_fault_detect` — 有潜伏故障

### 2.2 核心参数

| 参数 | 值 | 含义 |
|------|:---:|------|
| 配置接口 | 1 路 AXI4-Lite Slave | 用来配置安全岛 |
| 监控接口 | 5 路 AXI4 Read Master | 用来读外部寄存器 |
| 数据位宽 | 64 bit | 每次读 64 位 |
| 偏移地址 | 每路最多 64 个 | 每路 Master 最多监控 64 个寄存器 |
| Outstanding | 最多 4 个 | 同时可以有 4 个未完成的读请求 |

### 2.3 安全机制总览

```
┌─────────────────────────────────────────────────────┐
│ 检测层级                                            │
├─────────────────────────────────────────────────────┤
│ Layer 1: 系统故障 ── Mask+OR / Expected 比较 / CRC  │
│ Layer 2: 总线故障 ── Timeout / RRESP Error / RID    │
│ Layer 3: 配置故障 ── 非法参数 / 写保护违规           │
│ Layer 4: 内部故障 ── Shadow/反码 / TMR / 心跳 / KAT │
│ Layer 5: 潜在故障 ── Stuck-at 阈值 / 间歇性恢复     │
└─────────────────────────────────────────────────────┘
```

## 6. 信号调试技巧

### 6.1 ModelSim 波形关键信号

| 信号 | 含义 | 何时关注 |
|------|------|---------|
| `cfg_enable` | 安全岛使能 | 配置完成后应置 1 |
| `scan_busy` | 正在扫描 | 扫描过程中为 1 |
| `core_read_req[0:4]` | 读请求 | 每轮扫描每次置 1 |
| `m_axi_arvalid_flat[0:4]` | AR 通道 valid | 确认有读请求发出 |
| `axi_done_flat[0:4]` | 读完成 | 每次 RLAST 后 1 个脉冲 |
| `fault_detect` | 系统故障 | 期望有/无故障 |
| `safety_island_fault_detect` | 安全岛故障 | 注入内部故障后应置 1 |
| `fd_error_code` | 故障码 | `0x00`=无, `0x10`=配置, `0x20`=总线 |
| `heartbeat_active` | 心跳进行中 | 每 1024 cycle 激活一次 |

### 6.2 故障注入关键信号

| 信号 | 含义 |
|------|------|
| `+FAULT_INDEX=N` | 选择第 N 个预定义故障 |
| `+FAULT_KIND=xxx` | 指定注入目标种类 |
| `+BIT_INDEX=N` | 指定要翻转的 bit |
| `fi_force_now` | TB 内部：注入触发标志 |

## 7. 常见问题

### Q: 修改了 RTL 参数 (如 NUM_MASTERS) 后要改哪些？

A: 需要同步修改：
- `safety_island_top.v` 顶层 port 位宽
- `tb_safety_island_top_full.v` TB 中对应信号声明
- `axi_slave_mem_model.v` 存储模型
- ModelSim .do 脚本中的编译参数

### Q: CRC_WIDTH 从 8 改为 16 有什么影响？

A: 
- R 通道 `m_axi_rcheck_flat` 位宽从 `NUM_MASTERS*8` 变为 `NUM_MASTERS*16`
- CRC 多项式从 `0x07` 变为 `0x1021`
- 外部模块需要同步更新校验位生成逻辑
- 检测能力增强：从单 R beat 校验升级为 AR+R 双阶段 E2E

### Q: Icarus 和 ModelSim 结果不一致怎么办？

A: Icarus 已知有时序相关问题，以 ModelSim/VCS 结果为准。Icarus 主要用于快速开发迭代，正式验证用 ModelSim 或 VCS。

### Q: 如何增加新的注入目标？

A:
1. 在 `gen_fi_bit_list.py` 的 `TARGETS` 列表中添加新条目
2. 在 `tb_safety_island_fault_injection.v` 中添加对应的 `+FAULT_KIND` 处理逻辑
3. 在仿真中使用 `force` / `release` 注入故障
4. 需要对应的 `inv` 反码寄存器可访问（层次化路径匹配）
